# Public Shares

## 概要

- 目的: 配信用・外部共有用に、ログイン不要の閲覧専用URLを各機能で発行できるようにする
- 対象: `memo`, `todo`, `calendar`, `tool`, `event`
- 非対象: 参加型の共有、共同編集、公開ページ上での更新操作

## 設計

- 共有と公開は分離する
- 公開は `public_shares` テーブルで一元管理する
- URL は token ベースにし、公開ページは read-only に限定する
- カレンダーの既存 `share_link` は招待用として維持し、公開用とは混ぜない

## 実装方針

1. `public_shares` テーブルを追加
2. 認証ありAPIで公開リンクの作成・取得・削除を提供
3. 認証なしAPIで token から公開内容を取得
4. SPA に `public/:token` の read-only ページを追加
5. 各画面に共通の公開リンクダイアログを差し込む

## 今回のスコープ

- メモ: Markdown / table memo の公開
- ToDo: 単体ToDoの公開
- イベント: 単体予定の公開
- カレンダー: カレンダー本体と予定一覧の公開
- ツール: 単体ツールの公開
