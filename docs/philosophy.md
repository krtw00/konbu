# Philosophy — konbuの思想

## なぜ作ったか

メモ、ToDo、カレンダー、ブックマーク。日常的に使うツールがバラバラのサービスに散らばっている。Notionは重い。Obsidianはローカル専用。Google Calendarはデータを手元に置けない。どれも「自分の情報を自分で管理する」という要求を満たしてくれない。

konbuは「ブラウザを開いたらここから始まる、自分専用のワークスペース」として作った。

## 何を大事にしているか

### 自分のデータは自分の手元に

セルフホストで全機能が動く。外部サービスに依存しない。データはPostgreSQLに入っていて、JSON/Markdownでいつでもエクスポートできる。サービスが終わってもデータは消えない。

### 軽さ

起動が速い。ページ遷移が速い。入力を始めるまでの摩擦がゼロに近い。機能を足すことより、既にある機能が軽快に動くことを優先する。

### 検索できなければ保存する意味がない

情報は書いた瞬間より探す瞬間のほうが多い。メモ・ToDo・カレンダーをまたいだ横断検索を最重要機能として位置づけている。pg_bigmで日本語の全文検索にも対応する。

### APIが唯一の窓口

Web UIもCLIも外部連携も、すべて同じREST APIを通る。UIを持たない操作（CLIでのメモ追加、AIエージェント経由のタスク管理）も一級市民として扱う。

### 過剰に作らない

グループウェア機能（ワークフロー、設備予約、チャット）は対象外。個人のワークスペースに本当に必要なものだけを入れる。機能が少ないことは弱みではなく設計判断。

## 誰のためのツールか

- 自分のサーバーを持っている人
- 情報を1箇所に集約したい人
- CLIやAPIでデータにアクセスしたい人
- 重いツールに疲れた人

## konbuが目指さないもの

- リアルタイム共同編集
- WYSIWYGエディタ
- プラグインで無限に拡張できるプラットフォーム
- エンタープライズ向け機能
